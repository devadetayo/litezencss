Utility‑first & component‑versed—pixel‑perfect UIs, no code‑burst.

Litezen fuses utility‑first classes with ready‑made, accessible components so you can build pixel‑perfect, responsive UIs directly in your markup. Enjoy built‑in theming, state variants, and a performance‑first architecture—zero runtime overhead, all flow.

Start Building View Docs 

  <form class="lz-form max-w-md mx-auto">
    <div class="lz-form-group">
      <label class="lz-form-label">Username</label>
      <input type="text" class="lz-form-input" placeholder="johndoe"/>
    </div>
    <div class="lz-form-group">
      <label class="lz-form-label">Password</label>
      <input type="password" class="lz-form-input" placeholder="••••••"/>
    </div>
    <div class="lz-form-group flex items-center gap-2">
      <label class="lz-checkbox flex items-center gap-2">
        <input type="checkbox" class="lz-checkbox-input"/>
        <span class="lz-checkbox-box"></span>
        <span class="lz-checkbox-label">Remember me</span>
      </label>
    </div>
    <button type="submit" class="lz-button bg-primary text-white px-4 py-2 rounded">
      Log In
    </button>
  </form>

Why Litezen?

Utility‑First

Compose your entire design directly in HTML with single‑purpose classes like .p-4, .text-center and .bg-primary, eliminating custom CSS clutter.

Component‑Ready

Jump‑start your UI with prebuilt, theme‑aware components like .button, .card and .modal, all styled consistently out of the box.

Dash‑Variants

Effortlessly add responsive and state styles with prefixes like sm-, hover- and dark-, for fine‑grained control without extra CSS.

Theme‑Ready

Built‑in CSS tokens and a data‑driven toggle make light/dark mode a breeze—and you can extend to custom themes in minutes.

Accessibility‑First

All components include ARIA roles, focus‑styles, and keyboard support by default—so your UI is inclusive from day one.

Performance‑First

Zero JavaScript runtime, a small CSS footprint, and purgeable utilities ensure your bundles stay lean and load lightning‑fast.
